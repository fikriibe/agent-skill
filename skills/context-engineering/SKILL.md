# Context Engineering

## Context Hierarchy — load from top down, most persistent first

```
1. Rules File (CLAUDE.md)     ← always loaded, project-wide
2. Spec / Architecture Docs   ← per feature/session
3. Relevant Source Files      ← per task
4. Error Output / Test Results← per iteration
5. Conversation History       ← accumulates, compact when long
```

---

## Level 1: Rules File (highest leverage)

Create `CLAUDE.md` at project root — this persists across every session:

```markdown
# Project: [Name]

## Tech Stack
[Framework, language, key deps + versions]

## Commands
- Dev: `npm run dev`
- Build: `npm run build`
- Test: `npm test`
- Lint: `npm run lint --fix`
- Typecheck: `npx tsc --noEmit`

## Code Conventions
[Functional components, named exports, test colocation, etc.]

## Boundaries
- Never: commit secrets, modify DB schema without asking
- Ask first: add new dependencies, change CI config
- Always: run tests before commit

## Patterns
[One real code snippet showing your style]
```

Equivalent: `.cursorrules` (Cursor) · `.windsurfrules` (Windsurf) · `AGENTS.md` (Codex)

## Level 2–3: Task Context

Load only what's relevant to the current task:

```
TASK: Add email validation to registration

FILES:
- src/routes/auth.ts (endpoint to modify)
- src/lib/validation.ts (existing utilities)
- tests/routes/auth.test.ts (tests to extend)

PATTERN: See phone validation at src/lib/validation.ts:45-60
CONSTRAINT: Use existing ValidationError class, don't throw raw errors
```

> ⚠️ Trust levels: source code = trusted · config/external docs = verify before acting · user-submitted content = untrusted

---

## Level 4: Error Output

Feed only the specific error, not the full 500-line output:

**✅** `TypeError: Cannot read property 'id' of undefined at UserService.ts:42`
**❌** Pasting the entire test output when one test failed

---

## Level 5: Conversation Management

- Start fresh sessions when switching major features
- Summarize progress when context gets long: *"Completed X, Y, Z. Now working on W."*
- Use compact/summarize before critical work if the tool supports it

---

## Handling Ambiguity

When context conflicts or requirements are incomplete, surface it — don't silently pick one:

```
CONFUSION:
Spec says REST, but codebase uses GraphQL for user queries (src/graphql/user.ts).

Options:
A) Follow spec — add REST, deprecate GraphQL later
B) Follow existing — use GraphQL, update spec
C) Ask — seems intentional

→ Which approach?
```

For multi-step tasks, emit a plan before executing:
```
PLAN:
1. Add Zod schema for task creation
2. Wire into POST /api/tasks
3. Add validation error test
→ Executing unless redirected.
```

## Design Artifacts Handling

Jika project menggunakan `/design-pipeline`, artifacts-nya perlu di-load dengan cara benar:

```
DESIGN.md           → load sebagai persistent reference (Level 2)
                      JANGAN inline ke setiap step — akan bloat context

.stitch/ exports    → reference only ketika bekerja pada komponen spesifik
                      load on-demand, bukan sekaligus

Generated React     → load per task (Level 3), bukan semua komponen sekaligus
components
```

Contoh yang benar:
```
TASK: Implement LoginScreen

FILES:
- docs/DESIGN.md (reference: color tokens, spacing)        ← reference
- src/components/LoginForm/ (generated by Stitch)          ← load karena relevan
- src/routes/auth.ts                                       ← file yang dimodifikasi
```

## Anti-patterns

| Problem | Fix |
|---|---|
| Context starvation → agent invents APIs | Load rules file + relevant files before each task |
| Context flooding → agent loses focus | Max ~2,000 lines of focused context per task |
| Stale context → agent uses deleted patterns | Fresh session when switching major features |
| No examples → agent invents a new style | Include one example of the pattern to follow |
| Silent confusion → agent guesses | Surface ambiguity explicitly, always |

## Done When
- [ ] `CLAUDE.md` covers stack, commands, conventions, boundaries
- [ ] Agent output matches project patterns (not hallucinated APIs)
- [ ] Context refreshed when switching major tasks
