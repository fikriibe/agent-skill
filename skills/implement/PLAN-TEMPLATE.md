> Edit only between waves (or in `--review` mode). During execution, only `[ ]` → `[x]` flips are allowed.

## How to read this

- Role sections (`## BE Tasks`, `## FE Tasks`, etc.) — emitted only if work exists for that role
- Task IDs (`[BE-1]`, `[FE-1]`) — stable references across roles, used for `Depends:`
- `Depends:` — task IDs that must finish before this task starts. Empty = ready immediately.
- `Files:` — every path the subagent will touch. No overlap between tasks in the same wave.
- `Verify:` — the exact command that proves the task is done. Must pass before `[x]`.
- Dev tasks (FE/BE) write the failing test first, then implementation (TDD intrinsic).
